    Créer la route du formulaire

    On crée une route chargée d'afficher le formulaire. 

    from flask import render_template

@app.route("/books/create")
def create_book():
    return render_template("create-book.html")

    Lorsque l'utilisateur consulte l'adresse /books/create, Flask affiche le template create-book.html.

    Il faut donc créer ce fichier dans le dossier templates : 

    my_flask_project/
│
├── templates/
│   ├── books.html
│   └── create-book.html
│
└── app.py

    Créer la balise form

    Deux attributs contrôlent l'envoi du formulaire :

    Attribut Rôle
    action Indique l'adresse qui recevra les données
    method Indique la méthode HTTP utilisée pour envoyer les données

    Pour envoyer les données à la route create_book, on utilise url_for() : 

    <form
    action="{{ url_for('create_book') }}"
    method="post"
>

</form>

    La méthode post est utilisée lorsque le formulaire envoie des données destinées à créer ou modifier une ressource.

    L'attribut action ne contient pas directement l'adresse de la route. Il est généré à partir du nom de la fonction de vue.

    Ajouter les champs

    Chaque donnée demandée à l'utilisateur correspond à un champ du formulaire.

    Ajouter un champ de texte

    Le titre du livre peut être saisi avec un champ input de type text . 

    <label for="title">Titre</label>

<input
    type="text"
    id="title"
    name="title"
>

    La balise label indique à l'utilisateur la donnée attendue.

    Son attribut for doit correspondre à l'attribut id du champ.

    Ajouter un champ numérique

    L'année de parution peut être saisie avec un champ de type number. 

    <label for="year">Année de parution</label>

<input
    type="number"
    id="year"
    name="year"
>

    Ajouter une zone de texte

    Pour une donnée plus longue, comme le synopsis, utiliser la balise <textarea>. 

    <label for="synopsis">Synopsis</label>

<textarea
    id="synopsis"
    name="synopsis"
></textarea>

    Contrairement à input, la balise textarea possède une balise fermante.

    Ajouter le bouton d'envoi

    Un bouton de type submit permet d'envoyer le formulaire. 

    <button type="submit">
    Ajouter le livre
</button>

    Configurer les champs

    Plusieurs attributs HTML permettent de préciser les valeurs attendues.

    Attribut Rôle
    name Définit le nom utilisé lors de l'envoi de la valeur
    required Rend le champ obligatoire dans le navigateur
    maxlength Limite le nombre de caractères
    min Définit la plus petite valeur autorisée
    max Définit la plus grande valeur autorisée
    placeholder Affiche une indication temporaire dans le champ

    Comprendre l'attribut name

    L'attribut name est indispensable. Il identifie la valeur lorsque le formulaire est envoyé à Flask. 

    <input
    type="text"
    id="title"
    name="title"
>

    Dans cet exemple :

    • id="title" relie le champ à son label
    • name="title" donne un nom à la valeur envoyée

    Rendre les champs obligatoires

    <input
    type="text"
    id="title"
    name="title"
    maxlength="100"
    required
>

    L'attribut required empêche l'envoi du formulaire lorsque le champ est vide.

    L'attribut maxlength="100" correspond ici à la longueur définie dans le modèle avec db.String(100).

    Limiter l'année de parution

    <input
    type="number"
    id="year"
    name="year"
    min="0"
    max="2100"
    required
>

    Ces limites contrôlent la saisie dans le navigateur. Elles ne remplacent toutefois pas les vérifications réalisées dans l'application Flask.

    Erreurs fréquentes

    • Oublier l'attribut name : la valeur du champ n'est pas transmise à Flask.
    • Utiliser des valeurs différentes pour for et id : le label n'est pas correctement associé au champ.
    • Utiliser method="get" pour créer une donnée : les valeurs du formulaire apparaissent dans l'adresse de la page.
    • Oublier type="submit" sur le bouton : le comportement d'envoi peut ne pas être explicite.
    • Oublier la balise fermante de textarea : la structure HTML du formulaire devient incorrecte.
    • Faire confiance uniquement à l'attribut required : les données doivent également être vérifiées dans Flask.
    • Envoyer le formulaire vers une route qui n'accepte pas POST : Flask renvoie une erreur « Method Not Allowed ».
