    Créer le dossier static

    Les fichiers qui ne sont pas générés par Flask sont appelés fichiers statiques. Il peut notamment s'agir de feuilles de style CSS, d'images ou de fichiers JavaScript.

    À la racine du projet, créer un dossier nommé static. 

    my_flask_project/
│
├── .venv/
├── static/
├── templates/
│
├── app.py
└── requirements.txt

    Flask rend automatiquement les fichiers placés dans ce dossier accessibles à l'application.

    Ajouter une feuille de style

    Dans le dossier static, créer un dossier css, puis un fichier style.css. 

    static/
└── css/
    └── style.css

    Ajouter quelques règles dans le fichier CSS : 

    body {
    font-family: Arial, sans-serif;
    margin: 0;
}

header {
    padding: 20px;
}

main {
    max-width: 900px;
    margin: auto;
}

    Dans le fichier base.html, utiliser la fonction url_for() pour construire l'adresse de la feuille de style : 

    <link rel="stylesheet"
      href="{{ url_for('static', filename='css/style.css') }}">
    • L'argument static désigne le dossier des fichiers statiques.
    • Le paramètre filename indique le chemin du fichier à l'intérieur de ce dossier.

    Afficher une image

    Créer un dossier images dans le dossier static, puis y placer une image. 

    static/
├── css/
└── images/
    └── profile.jpg

    Dans un template, construire l'adresse de l'image avec url_for() : 

    <img
    src="{{ url_for('static', filename='images/profile.jpg') }}"
    alt="Photo de présentation"
>

    Le chemin indiqué dans filename commence à l'intérieur du dossier static.

    Ajouter un fichier JavaScript

    Les fichiers JavaScript sont aussi placés dans le dossier static.

    Créer un dossier js, puis un fichier script.js. 

    static/
├── css/
├── images/
└── js/
    └── script.js

    Dans le fichier JavaScript : 

    console.log("Le fichier JavaScript est chargé.");

    Puis, dans base.html, ajouter la balise suivante dans le <head> : 

    <script src="{{ url_for('static', filename='js/script.js') }}" defer></script>

    Erreurs fréquentes

    • Donner un autre nom au dossier static : Flask ne trouve pas automatiquement les fichiers.
    • Indiquer static/ au début de filename : le dossier static est déjà désigné par le premier argument.
    • Utiliser un chemin qui ne correspond pas à l'arborescence : le navigateur ne trouve pas le fichier.
    • Modifier un fichier sans actualiser le navigateur : l'ancienne version peut rester affichée.
