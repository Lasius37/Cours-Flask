    Comprendre le problème

    Après l'enregistrement d'un livre, la fonction de vue peut renvoyer directement un message : 

    db.session.add(book)
db.session.commit()

return "Le livre a été enregistré."

    Cette solution confirme l'enregistrement, mais elle ne dirige pas l'utilisateur vers une véritable page de l'application. Il serait plus utile de renvoyer l'utilisateur vers la liste des livres après l'enregistrement. Pour cela, Flask fournit deux fonctions :

    Fonction Rôle
    url_for() Génère l'URL correspondant à une fonction de vue
    redirect() Demande au navigateur de consulter une autre URL

    Générer une URL avec url_for()

    La fonction url_for() génère une URL à partir du nom d'une fonction de vue. Considérons une vue qui affiche tous les livres : 

    @app.route("/books")
def books():
    books = db.session.execute(
        db.select(Book)
    ).scalars().all()

    return render_template(
        "books.html",
        books=books
    )

    Le nom de cette fonction de vue est books. Son URL peut donc être générée ainsi : 

    url = url_for("books")

    La variable url contient alors : 

    /books

    Attention, il faut importer url_for depuis Flask : 

    from flask import url_for

    Il est préférable d'utiliser url_for() plutôt que d'écrire directement l'adresse : 

    url = "/books"  # adresse écrite directement (codée "en dur")

url = url_for("books")  # adresse générée à partir de la fonction de vue

    Si l'adresse de la route est modifiée plus tard, url_for() générera automatiquement la nouvelle URL.

    Rediriger avec redirect()

    La fonction redirect() crée une réponse demandant au navigateur de consulter une autre adresse. Elle doit également être importée depuis Flask : 

    from flask import redirect

    Une redirection vers la liste des livres peut être écrite ainsi : 

    return redirect("/books")

    Il est cependant préférable de combiner redirect() et url_for() : 

    return redirect(
    url_for("books")
)

    Le fonctionnement se déroule en deux étapes :

    • url_for("books") génère l'adresse /books
    • redirect() demande au navigateur de consulter cette adresse

    Appliquer le principe POST-Redirect-GET

    Après le traitement d'un formulaire envoyé avec POST, il est recommandé d'effectuer une redirection. Cette organisation est appelée POST-Redirect-GET.

    Étape Action
    POST Le formulaire envoie les données
    Redirect Flask demande au navigateur de consulter une autre page
    GET Le navigateur demande la page de destination

    Dans la vue create_book, le message provisoire renvoyé par return peut donc être remplacé.

    Avant : 

    db.session.add(book)
db.session.commit()

return "Le livre a été enregistré."

    Après : 

    db.session.add(book)
db.session.commit()

return redirect(
    url_for("books")
)

    Une fois le livre enregistré, l'utilisateur est redirigé vers la liste des livres. Cette redirection évite également qu'un rafraîchissement de la page renvoie immédiatement les mêmes données du formulaire.

    Transmettre des paramètres à url_for()

    Certaines routes contiennent une partie variable. 

    @app.route("/books/<int:book_id>")
def book_detail(book_id):
    book = db.session.get(Book, book_id)

    if book is None:
        return "Livre introuvable", 404

    return render_template(
        "book-detail.html",
        book=book
    )

    Pour générer l'URL de cette route, il faut transmettre une valeur pour book_id : 

    url = url_for(
    "book_detail",
    book_id=1
)

    L'URL générée est : 

    /books/1

    Après la création d'un livre, il serait donc possible de rediriger vers sa page de détail : 

    db.session.add(book)
db.session.commit()

return redirect(
    url_for(
        "book_detail",
        book_id=book.id
    )
)

    Après commit(), l'attribut book.id contient l'identifiant attribué au nouveau livre.

    Erreurs fréquentes

    • Oublier d'importer redirect ou url_for : Flask ne reconnaît pas la fonction utilisée.
    • Transmettre à url_for() l'adresse d'une route au lieu du nom de sa fonction : Flask ne trouve aucun endpoint correspondant.
    • Utiliser redirect("books") : le navigateur est dirigé vers une adresse relative incorrecte.
    • Oublier return devant redirect() : la réponse de redirection n'est pas renvoyée par la fonction.
    • Rediriger avant db.session.commit() : le livre peut ne pas être enregistré avant le changement de page.
    • Oublier un paramètre demandé par une route dynamique : url_for() ne peut pas construire l'URL.
    • Afficher directement une page après une requête POST sans redirection : un rafraîchissement peut proposer de renvoyer les données du formulaire.
