    Comprendre le rôle d'un modèle

    Un modèle décrit la structure des données que l'application doit enregistrer.

    Chaque modèle correspond généralement à une table de la base de données :

    • la classe Python représente la table
    • les attributs de la classe représentent les colonnes
    • les objets créés à partir de la classe représenteront les lignes

    Par exemple, pour une bibliothèque, le modèle Book pourra contenir :

    Colonne Contenu
    id Identifiant unique du livre
    title Titre du livre
    year L'année de parution
    synopsis Le synopsis du livre

    Créer un modèle

    Un modèle est une classe Python qui hérite de db.Model. 

    class Book(db.Model):
    pass

    Flask-SQLAlchemy transforme ensuite ce modèle en table dans la base de données. Lorsque le nom de la table n'est pas indiqué, il peut être généré automatiquement.

    Il est toutefois possible de définir explicitement ce nom avec __tablename__ : 

    class Book(db.Model):
    __tablename__ = "books"

    Le modèle Book correspondra ainsi à une table nommée books.

    Définir les colonnes

    Chaque colonne est créée avec db.Column(). 

    class Book(db.Model):
    __tablename__ = "books"
    id = db.Column(db.Integer)
    title = db.Column(db.String)
    year = db.Column(db.Integer)
    synopsis = db.Column(db.Text)

    Le premier argument indique le type de donnée que la colonne pourra contenir.

    Type Utilisation
    db.Integer Nombre entier
    db.String Texte court
    db.Text Texte long
    db.Boolean Valeur vraie ou fausse
    db.Float Nombre décimal

    La longueur maximale d'un texte court peut être précisée : 

    title = db.Column(db.String(100))

    La colonne title pourra ainsi contenir un texte court dont la longueur maximale prévue est de 100 caractères.

    Configurer les colonnes

    Plusieurs options permettent de préciser le fonctionnement d'une colonne.

    Option Rôle
    primary_key=True Définit la clé primaire de la table
    nullable=False Rend la valeur obligatoire
    unique=True Interdit les valeurs identiques
    default=... Définit une valeur par défaut

    Chaque modèle doit posséder une clé primaire permettant d'identifier chaque ligne de façon unique. 

    id = db.Column(
    db.Integer,
    primary_key=True
)

    La valeur de id sera générée automatiquement pour chaque nouveau livre.

    Le titre et le synopsis peuvent être rendus obligatoires : 

    title = db.Column(
    db.String(100),
    nullable=False
)

year = db.Column(
    db.Integer,
    nullable=False
)

synopsis = db.Column(
    db.Text,
    nullable=False
)

    Créer la table

    La création d'un modèle ne crée pas immédiatement la table correspondante dans la base de données.

    Après la définition du modèle, utiliser db.create_all() dans le contexte de l'application : 

    with app.app_context():
    db.create_all()

    Le contexte d'application permet à Flask-SQLAlchemy de savoir quelle application et quelle configuration doivent être utilisées. Lors du lancement du programme :

    • le dossier instance est créé si nécessaire
    • le fichier my_db_file.db est créé
    • la table books est ajoutée à la base

    La méthode create_all() crée uniquement les tables qui n'existent pas encore. Elle ne met pas à jour automatiquement la structure d'une table déjà créée.

    Erreurs fréquentes

    • Ne pas faire hériter le modèle de db.Model : Flask-SQLAlchemy ne reconnaît pas la classe comme un modèle.
    • Oublier la clé primaire : SQLAlchemy ne peut pas identifier correctement les lignes.
    • Oublier nullable=False sur une donnée obligatoire : une valeur vide pourra être enregistrée dans la colonne.
    • Appeler db.create_all() avant la définition du modèle : la table du modèle n'est pas encore connue.
    • Appeler db.create_all() sans contexte d'application : Flask-SQLAlchemy ne peut pas accéder à l'application courante.
    • Modifier un modèle après la création de la table et relancer create_all() : la table existante n'est pas automatiquement modifiée.
