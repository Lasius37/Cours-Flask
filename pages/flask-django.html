    Comprendre le rôle d'un framework web

    Une application web doit recevoir une requête envoyée par un navigateur, exécuter du code, puis construire une réponse. Cette réponse peut être une page HTML, une redirection, un fichier ou des données.

    Il serait possible de programmer entièrement ce fonctionnement avec Python. Cependant, il faudrait alors créer soi-même de nombreux outils : la gestion des URL, la lecture des formulaires, les réponses HTTP, les templates ou encore les sessions.

    Un framework web fournit une structure et des outils prêts à l'emploi pour faciliter la création d'une application web.

    Django et Flask sont deux frameworks web écrits en Python. Ils permettent donc de réaliser des applications similaires, mais ils ne proposent pas la même manière de construire un projet.

    Identifier les points communs

    Django et Flask reposent sur les mêmes principes généraux.

    • Ils utilisent Python pour programmer le fonctionnement de l'application.
    • Ils associent des URL à des fonctions ou à des classes Python (les vues).
    • Ils peuvent générer des pages HTML à partir de templates.
    • Ils permettent de recevoir les données envoyées par un formulaire.
    • Ils peuvent communiquer avec une base de données.
    • Ils peuvent gérer des comptes utilisateurs, des sessions et des fichiers.
    • Ils séparent le fonctionnement de l'application de sa présentation HTML et CSS.
    • Ils traitent les requêtes avec un schéma analogue.

    Comparer les deux approches

    Flask : un framework léger et modulable Django : un framework complet

    Flask fournit un noyau plus réduit. Une application minimale peut être créée dans un seul fichier Python.

    Le développeur ajoute ensuite uniquement les outils dont son projet a besoin. Par exemple, il peut ajouter une bibliothèque pour la base de données, les formulaires ou l'authentification.

    Flask impose moins de décisions. Le développeur dispose donc de davantage de liberté, mais il doit aussi choisir lui-même l'organisation et les bibliothèques de son application.

    Django fournit dès son installation une organisation de projet et de nombreuses fonctionnalités.

    • une structure de projet prédéfinie
    • un système de configuration
    • un outil de gestion des URL
    • un moteur de templates
    • un ORM pour manipuler les bases de données
    • un système de migrations
    • une interface d'administration
    • un système d'authentification
    • des outils pour créer et valider des formulaires

    Le développeur doit respecter les conventions de Django. En contrepartie, il dispose rapidement d'un environnement complet et cohérent.

    Choisir entre Django et Flask

    Utiliser plutôt Django

    Django est particulièrement adapté lorsqu'une application nécessite rapidement de nombreuses fonctionnalités déjà intégrées.

    Flask est particulièrement adapté lorsqu'on souhaite construire progressivement une application et contrôler précisément les outils utilisés.

    Utiliser plutôt Flask Utiliser plutôt Django
    • un petit site dynamique
    • un prototype
    • une application web simple
    • une API
    • un projet pédagogique
    • un projet nécessitant une organisation personnalisée
    • une application de gestion comportant plusieurs modèles
    • une interface d'administration
    • un système complet de comptes utilisateurs
    • de nombreux formulaires
    • une organisation commune pour une équipe de développeurs
    • un projet important destiné à évoluer

    Ce choix n'est pas déterminé uniquement par la taille du projet. Une application Flask peut devenir complexe et une petite application peut être créée avec Django. Le choix dépend surtout des besoins du projet et de la manière dont le développeur souhaite travailler.

    Éviter les idées reçues

    • Penser que Flask ne permet de créer que de petits projets : une application Flask peut être organisée en plusieurs modules et devenir complexe.
    • Penser que Django réalise automatiquement toute l'application : le développeur doit toujours définir les données, les vues, les templates et les règles de fonctionnement.
    • Penser que Flask est toujours plus facile : il est simple pour commencer, mais demande davantage de choix lorsque le projet grandit.
    • Penser qu'il faut obligatoirement choisir le framework le plus puissant : il faut choisir l'outil adapté aux besoins du projet.
    • Mélanger Flask et Django dans une même application : ils répondent au même besoin général et sont normalement utilisés séparément.
