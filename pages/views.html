    Distinguer une route et une vue

    Une route associe une adresse de l'application à une fonction Python. Cette fonction est appelée une vue. Elle est exécutée lorsque l'utilisateur ouvre l'adresse correspondante et renvoie la réponse à afficher. 

    @app.route("/")                # route
def accueil():                 # vue
    return "Bienvenue sur la page d'accueil"
    • @app.route("/") définit l'adresse de la page
    • accueil() est la vue exécutée par Flask
    • return fournit la réponse envoyée au navigateur

    Créer plusieurs routes et vues

    Une application contient généralement plusieurs routes. Chaque route appelle une vue différente. 

    @app.route("/")
def accueil():
    return "Bienvenue sur la page d'accueil"

@app.route("/about")
def about():
    return "À propos de ce site"

@app.route("/contact")
def contact():
    return "Page de contact"

    Le nom de la fonction ne détermine pas l'adresse de la page. C'est le chemin écrit dans @app.route() qui définit l'URL.

    Créer une route dynamique

    Une route dynamique contient une valeur qui peut changer. Cette valeur est placée entre chevrons dans l'adresse. Le nom placé entre les chevrons doit être identique au nom du paramètre de la fonction. 

    @app.route("/user/<name>")
def user(name):
    return f"Bienvenue {name}"

    La valeur récupérée dans l'adresse est transmise au paramètre name de la vue :

    • /user/Pierre affiche Bienvenue Pierre
    • /user/Alice affiche Bienvenue Alice

    Utiliser les convertisseurs

    Par défaut, la valeur récupérée dans une route dynamique est une chaîne de caractères. Un convertisseur permet d'imposer un autre type de valeur. 

    @app.route("/article/<int:article_id>")
def article(article_id):
    return f"Article numéro {article_id}"

    Le convertisseur int indique que article_id doit être un nombre entier.

    • /article/12 correspond à la route
    • /article/python ne correspond pas à la route et provoque une erreur 404
    Convertisseur Valeur acceptée Exemple
    string texte sans caractère / <string:name>
    int nombre entier positif <int:id>
    float nombre décimal positif <float:price>
    path texte pouvant contenir des caractères / <path:subpath>
    uuid identifiant au format UUID <uuid:user_id>

    Le convertisseur string est utilisé par défaut. Il n'est donc généralement pas nécessaire de l'indiquer. Les deux routes suivantes sont donc identiques : 

    @app.route("/user/<name>")

@app.route("/user/<string:name>")

    Renvoyer une réponse

    Une vue doit toujours renvoyer une réponse avec return. Pour le moment, les vues renvoient simplement du texte : 

    @app.route("/contact")
def contact():
    return "Page de contact"

    Il est également possible de renvoyer directement du code HTML : 

    @app.route("/about")
def about():
    return "<h1>À propos</h1>"

    Cette méthode devient alors rapidement difficile à lire et elle est inadaptée. Les pages HTML seront donc placées dans des fichiers séparés appelés templates.

    Erreurs fréquentes

    • Oublier le caractère / au début du chemin : la route n'est pas correctement déclarée.
    • Utiliser deux fois le même nom de fonction : Flask ne peut pas distinguer les deux vues.
    • Ouvrir une adresse qui ne correspond à aucune route : le navigateur affiche une erreur 404.
    • Utiliser un nom différent dans la route et dans le paramètre de la vue : la valeur dynamique ne peut pas être transmise correctement.
    • Fournir une valeur qui ne correspond pas au convertisseur : la route n'est pas reconnue et le navigateur affiche une erreur 404.
    • Oublier l'instruction return : la vue ne renvoie aucune réponse.
