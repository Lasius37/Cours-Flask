    Créer l'application

    À la racine du projet, créer un fichier Python nommé app.py. Ce fichier contiendra le code principal de la première application Flask. Le projet devient : 

    my_flask_project/
│
├── .venv/
│
├── app.py                  application Flask
└── requirements.txt

    Dans ce fichier app.py, importer la classe Flask depuis la bibliothèque flask. 

    from flask import Flask

    Créer ensuite une instance de la classe Flask et l'enregistrer dans une variable nommée app. 

    app = Flask(__name__)

    La variable app représente l'application Flask. Elle permettra notamment de définir les différentes pages du site.

    La variable spéciale __name__ indique à Flask dans quel fichier l'application est créée.

    Créer une première page

    Une page Flask est associée à une adresse grâce au décorateur @app.route(). 

    @app.route("/")
def accueil():
    return "Bienvenue sur mon application Flask !"

    La valeur "/" représente la racine du site, c'est-à-dire sa page d'accueil. Lorsque cette adresse est demandée, Flask exécute la fonction accueil().

    Le texte placé après return est ensuite envoyé au navigateur.

    L'application est créée, mais elle n'est pas encore exécutée.

    Démarrer le serveur

    Dans le terminal de PyCharm, lancer le serveur de développement avec la commande : 

    flask run

    Flask trouve automatiquement l'application lorsque le fichier est nommé app.py et que l'instance de l'application est enregistrée dans une variable nommée app.

    Pendant le développement, il est préférable d'activer le mode debug : 

    flask run --debug

    Ce mode redémarre automatiquement le serveur lorsque le code Python est modifié. Il affiche également davantage d'informations lorsqu'une erreur se produit. Le mode debug doit uniquement être utilisé pendant le développement de l'application.

    Lorsque le serveur démarre, le terminal affiche notamment : 

    * Serving Flask app 'app'
* Running on http://127.0.0.1:5000

    Tant que le serveur fonctionne, le terminal reste occupé et affiche les requêtes reçues par l'application.

    Pour arrêter le serveur, sélectionner le terminal et utiliser : 

    Ctrl + C

    Ouvrir l'application

    Quand le serveur est lancé, dans un navigateur, ouvrir l'adresse indiquée dans le terminal : 

    http://127.0.0.1:5000
    • L'adresse 127.0.0.1 désigne l'ordinateur sur lequel l'application est exécutée.
    • Le nombre 5000 correspond au port utilisé par défaut par le serveur Flask.

    Le navigateur doit afficher le message défini dans la fonction accueil() : 

    Bienvenue sur mon application Flask !

    Erreurs fréquentes

    • Nommer le fichier flask.py : Python risque de confondre ce fichier avec la bibliothèque Flask.
    • Écrire Flask sans majuscule : le nom de la classe doit être respecté exactement.
    • Lancer le serveur depuis un autre dossier : Flask ne trouve pas le fichier app.py.
    • Lancer le mode debug en production : risque important de sécurité.
    • Utiliser un terminal dans lequel l'environnement virtuel n'est pas activé : la commande Flask peut être introuvable.
    • Utiliser un port déjà occupé : le serveur ne peut pas démarrer sur ce port.
