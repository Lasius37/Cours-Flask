    Créer une donnée

    Une ligne de la table est représentée en Python par un objet créé à partir du modèle.

    Pour créer un nouveau livre, instancier la classe Book en indiquant la valeur de chaque colonne : 

    book = Book(
    title="Le Petit Prince",
    year=1943,
    synopsis="Un aviateur rencontre un jeune prince venu d'une autre planète..."
)

    Il n'est pas nécessaire d'indiquer la valeur de id, car celle-ci sera générée automatiquement.

    À ce stade, l'objet existe uniquement dans le programme Python. Il n'est pas encore enregistré dans la base de données.

    Enregistrer une donnée

    Pour préparer l'enregistrement d'un objet, il faut l'ajouter à la session : 

    db.session.add(book)

    La session regroupe les modifications qui doivent être appliquées à la base de données.

    Ensuite, on valide ces modifications : 

    db.session.commit()

    L'enregistrement complet nécessite donc trois étapes : 

    book = Book(  # création de l'enregistrement
    title="Le Petit Prince",
    year=1943,
    synopsis="Un aviateur rencontre un jeune prince venu d'une autre planète..."
)

db.session.add(book)  # ajout de l'enregistrement à la session

db.session.commit()  # enregistrement de la session dans la base

    Après l'appel de commit(), le livre est enregistré durablement dans la table books.

    Pour effectuer cette opération directement au lancement du programme, utiliser le contexte de l'application : 

    with app.app_context():
    book = Book(
        title="Le Petit Prince",
        year=1943,
        synopsis="Un aviateur rencontre un jeune prince venu d'une autre planète."
    )

    db.session.add(book)

    db.session.commit()

    Récupérer toutes les données

    Pour récupérer des données, construire une requête avec db.select(). 

    query = db.select(Book)

    Cette requête indique que l'application souhaite sélectionner les objets du modèleBook.

    Exécuter ensuite la requête avec la session : 

    result = db.session.execute(query)

    La méthode scalars() récupère les objets Book, puis all() les rassemble dans une liste : 

    books = db.session.execute(
    db.select(Book)
).scalars().all()

    Les livres peuvent également être triés (ici, par titre) : 

    books = db.session.execute(
    db.select(Book).order_by(Book.title)
).scalars().all()

    Récupérer une donnée

    Une donnée peut être retrouvée à partir de sa clé primaire : 

    book = db.session.get(Book, 1)

    Cette instruction recherche dans la table books le livre dont l'identifiant vaut 1.

    Si aucun livre ne possède cet identifiant, la valeur None est renvoyée. 

    if book is None:
    return "Livre introuvable"

    Une donnée peut aussi être recherchée à partir de la valeur d'une colonne : 

    book = db.session.execute(
    db.select(Book).filter_by(
        title="Le Petit Prince"
    )
).scalar_one_or_none()

    La méthode scalar_one_or_none() renvoie :

    • l'objet trouvé lorsqu'une seule donnée correspond
    • None lorsqu'aucune donnée ne correspond
    • une erreur si plusieurs données correspondent

    Afficher les données

    Pour afficher les livres dans une page, on les récupère dans la vue puis on les transmet au template avec render_template(). 

    from flask import Flask, render_template

@app.route("/books")
def books():
    books = db.session.execute(
        db.select(Book).order_by(Book.title)
    ).scalars().all()

    return render_template(
        "books.html",
        books=books
    )

    Dans le fichier templates/books.html, on parcourt la liste avec une boucle Jinja : 

    <h1>Bibliothèque</h1>

{% if books %}
    {% for book in books %}
        <article>
            <h2>{{ book.title }} ({{ book.year }}) </h2>
            <p>{{ book.synopsis }}</p>
        </article>
    {% endfor %}
{% else %}
    <p>Aucun livre enregistré.</p>
{% endif %}

    Les colonnes du modèle sont accessibles dans le template avec la notation par point :

    • book.id
    • book.title
    • book.year
    • book.synopsis

    Erreurs fréquentes

    • Créer un objet sans l'ajouter à la session : la donnée n'est pas préparée pour être enregistrée.
    • Oublier db.session.commit() : les modifications ne sont pas enregistrées durablement.
    • Ajouter le même livre au lancement de chaque programme : une nouvelle ligne identique est créée à chaque exécution.
    • Oublier scalars() : le résultat contient des lignes SQL plutôt que directement les objets.
    • Utiliser un nom de variable différent dans la vue et le template : la liste ne peut pas être affichée correctement.
    • Utiliser scalar_one_or_none() alors que plusieurs résultats peuvent correspondre : SQLAlchemy signale que plusieurs lignes ont été trouvées.
