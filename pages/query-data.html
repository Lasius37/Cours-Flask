    Récupérer toutes les données

    Pour récupérer des données, on construit une requête avec db.select(). 

    query = db.select(Book)

    Cette requête indique que l'application souhaite sélectionner les objets du modèle Book. On exécute ensuite la requête avec la session : 

    result = db.session.execute(query)

    La méthode scalars() extrait les objets Book du résultat.

    La méthode all() rassemble ensuite tous les objets dans une liste : 

    books = db.session.execute(
    db.select(Book)
).scalars().all()

    La variable books contient désormais une liste de tous les livres enregistrés dans la table.

    Récupérer une donnée

    Une donnée peut être retrouvée à partir de sa clé primaire avec db.session.get(). 

    book = db.session.get(Book, 1)

    Cette instruction recherche dans la table books le livre dont l'identifiant vaut 1. Si aucun livre ne possède cet identifiant, la valeur None est renvoyée. 

    if book is None:
    return "Livre introuvable"

    Il est également possible de rechercher un objet à partir de la valeur d'une colonne : 

    book = db.session.execute(
    db.select(Book).filter_by(
        title="Le Petit Prince"
    )
).scalar_one_or_none()

    La méthode scalar_one_or_none() renvoie :

    • l'objet trouvé lorsqu'une seule donnée correspond
    • None lorsqu'aucune donnée ne correspond
    • une erreur lorsque plusieurs données correspondent

    Cette méthode est donc adaptée lorsque la requête doit normalement renvoyer au maximum un seul résultat.

    Filtrer les données

    Rechercher une valeur précise

    La méthode filter_by() permet de rechercher une valeur précise dans une colonne. 

    books = db.session.execute(
    db.select(Book).filter_by(
        year=1943
    )
).scalars().all()

    Cette requête récupère tous les livres dont l'année de parution est exactement 1943.

    Effectuer une comparaison

    La méthode where() permet d'utiliser des opérateurs de comparaison. 

    books = db.session.execute(
    db.select(Book).where(
        Book.year >= 1950
    )
).scalars().all()

    Cette requête récupère tous les livres publiés à partir de 1950.

    Opérateur Signification
    == Égal à
    != Différent de
    > Supérieur à
    >= Supérieur ou égal à
    < Inférieur à
    <= Inférieur ou égal à

    Utiliser plusieurs conditions

    Plusieurs conditions peuvent être placées dans la méthode where() . 

    books = db.session.execute(
    db.select(Book).where(
        Book.year >= 1940,
        Book.year <= 1960
    )
).scalars().all()

    Cette requête récupère les livres publiés entre 1940 et 1960. Toutes les conditions doivent être respectées.

    Rechercher dans un texte

    La méthode contains() permet de rechercher une suite de caractères dans une colonne textuelle. 

    books = db.session.execute(
    db.select(Book).where(
        Book.title.contains("Prince")
    )
).scalars().all()

    Cette requête récupère les livres dont le titre contient le mot Prince.

    Trier les données

    La méthode order_by() permet d'ordonner les résultats d'une requête.

    Effectuer un tri croissant

    Pour trier les livres par titre dans l'ordre alphabétique : 

    books = db.session.execute(
    db.select(Book).order_by(
        Book.title
    )
).scalars().all()

    Par défaut, le tri est croissant. Il est également possible de rendre ce choix explicite avec asc() : 

    books = db.session.execute(
    db.select(Book).order_by(
        Book.year.asc()
    )
).scalars().all()

    Les livres sont alors classés du plus ancien au plus récent.

    Effectuer un tri décroissant

    La méthode desc() permet d'inverser l'ordre du tri. 

    books = db.session.execute(
    db.select(Book).order_by(
        Book.year.desc()
    )
).scalars().all()

    Les livres sont alors classés du plus récent au plus ancien.

    Trier selon plusieurs colonnes

    Plusieurs critères de tri peuvent être indiqués dans order_by(). 

    books = db.session.execute(
    db.select(Book).order_by(
        Book.year.asc(),
        Book.title.asc()
    )
).scalars().all()

    Les livres sont d'abord classés par année, puis par titre lorsque plusieurs livres possèdent la même année.

    Combiner filtres et tris

    Une requête peut contenir plusieurs opérations successives. 

    books = db.session.execute(
    db.select(Book)
        .where(Book.year >= 1940)
        .order_by(Book.year.desc())
).scalars().all()

    Cette requête :

    • sélectionne les livres publiés à partir de 1940
    • les classe du plus récent au plus ancien
    • renvoie les résultats sous la forme d'une liste d'objets

    Il est souvent plus lisible de construire la requête en plusieurs étapes : 

    query = db.select(Book)

query = query.where(
    Book.year >= 1940
)

query = query.order_by(
    Book.year.desc()
)

books = db.session.execute(
    query
).scalars().all()

    Cette écriture facilite la lecture et permet d'ajouter progressivement de nouveaux filtres ou critères de tri.

    Afficher les données

    Pour afficher les livres dans une page, les récupérer dans une vue puis les transmettre au template avec render_template(). 

    from flask import Flask, render_template

@app.route("/books")
def books():
    books = db.session.execute(
        db.select(Book).order_by(
            Book.year.desc()
        )
    ).scalars().all()

    return render_template(
        "books.html",  # le template cible
        books=books  # Les données transmises
    )

    La variable books est transmise au template sous le même nom.

    Dans le fichier templates/books.html, on pacourt la liste avec une boucle Jinja : 

    <h1>Bibliothèque</h1>

{% if books %}
    {% for book in books %}
        <article>
            <h2>
                {{ book.title }} ({{ book.year }})
            </h2>

            <p>{{ book.synopsis }}</p>
        </article>
    {% endfor %}
{% else %}
    <p>Aucun livre enregistré.</p>
{% endif %}

    Les colonnes du modèle sont accessibles dans le template avec la notation par point :

    • book.id
    • book.title
    • book.year
    • book.synopsis

    Les filtres et les tris sont effectués dans la vue avant l'envoi des données au template.

    Erreurs fréquentes

    • Oublier scalars() : le résultat contient des lignes SQL plutôt que directement les objets.
    • Oublier all() lors de la récupération de plusieurs données : le résultat n'est pas transformé en liste.
    • Utiliser scalar_one_or_none() alors que plusieurs résultats peuvent correspondre : SQLAlchemy signale que plusieurs lignes ont été trouvées.
    • Utiliser filter_by() pour effectuer une comparaison : filter_by() est surtout adapté à la recherche d'une valeur précise.
    • Oublier d'utiliser les attributs du modèle dans where() : la condition ne correspond pas à une colonne de la table.
    • Confondre asc() et desc() : les résultats sont affichés dans l'ordre inverse de celui attendu.
    • Utiliser un nom de variable différent dans la vue et dans le template : les données ne peuvent pas être affichées correctement.
    • Appliquer les filtres uniquement dans le template : des données inutiles sont récupérées et la logique devient plus difficile à maintenir.
