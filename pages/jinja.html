    Du Python dans le HTML

    Un template HTML peut contenir des informations fixes, mais aussi des données produites par l'application Flask. Flask utilise le moteur de templates Jinja pour intégrer ces données dans les pages HTML en Python. Cela suppose une syntaxe un peu particulière. Jinja utilise principalement trois types de délimiteurs :

    • {{ ... }} pour afficher une donnée ;
    • {% ... %} pour exécuter une instruction ;
    • {# ... #} pour écrire un commentaire Jinja. 
    <h1>{{ title }}</h1>

{% if user %}
    <p>Bienvenue !</p>
{% endif %}

{# Ce commentaire n'apparaît pas dans la page HTML. #}

    Transmettre une donnée au template

    Les données sont transmises au template depuis la vue avec la fonction render_template(). 

    @app.route("/")
def accueil():
    return render_template(
        "index.html",
        title="Accueil"
    )

    Dans cet exemple, la donnée "Accueil" est transmise au template sous le nom title. La syntaxe utilisée est : 

    render_template(
    "my-template.html",
    my_variable=value
)

    Afficher une donnée

    Dans le template, placer le nom de la variable entre doubles accolades : 

    <h1>{{ title }}</h1>

    Lors de l'affichage de la page, Jinja remplace la variable par sa valeur : 

    <h1>Accueil</h1>

    Une variable peut être utilisée dans un n'importe quel emplacement du template. 

    <title>{{ title }}</title>
<p>Bonjour {{ firstname }}.</p>
<img src="{{ image }}" alt="Photo de présentation">

    Transmettre plusieurs données

    Une vue peut transmettre plusieurs données au même template. 

    @app.route("/profile")
def profile():
    return render_template(
        "profile.html",
        title="Mon profil",
        firstname="Alex",
        age=15,
        city="Lyon"
    )

    Chaque donnée est ensuite accessible avec le nom choisi dans la vue : 

    <h1>{{ title }}</h1>
<p>Prénom : {{ firstname }}</p>
<p>Âge : {{ age }} ans</p>
<p>Ville : {{ city }}</p>

    Il est souvent plus judicieux de regrouper plusieurs informations dans un dictionnaire. 

    context = {
    "firstname": "Alex",
    "age": 15,
    "city": "Lyon"
}
return render_template(
    "profile.html",
    context=context
)

    Les valeurs du dictionnaire sont accessibles avec un point : 

    <p>Prénom : {{ context.firstname }}</p>
<p>Âge : {{ context.age }} ans</p>
<p>Ville : {{ context.city }}</p>

    Utiliser des conditions

    Une condition permet d'afficher un contenu uniquement lorsque certaines conditions sont respectées. 

    @app.route("/profile")
def profile():
    return render_template(
        "profile.html",
        show_city=True,
        city="Lyon"
    )

    Dans le template : 

    {% if show_city %}
    <p>Ville : {{ city }}</p>
{% endif %}

    Une alternative peut être ajoutée avec else. 

    {% if show_city %}
    <p>Ville : {{ city }}</p>
{% else %}
    <p>La ville n'est pas renseignée.</p>
{% endif %}

    Plusieurs cas peuvent être testés avec elif. 

    {% if age >= 18 %}
    <p>Utilisateur majeur.</p>
{% elif age >= 13 %}
    <p>Utilisateur adolescent.</p>
{% else %}
    <p>Utilisateur enfant.</p>
{% endif %}

    Parcourir une liste

    Une boucle permet de répéter une partie du HTML pour chaque élément d'une liste. 

    @app.route("/skills")
def skills():
    skills = ["HTML", "CSS", "JavaScript"]

    return render_template(
        "skills.html",
        skills=skills
    )

    Dans le template, la boucle for génère un élément de liste pour chaque compétence. 

    <ul>
    {% for skill in skills %}
        <li>{{ skill }}</li>
    {% endfor %}
</ul>

    Jinja permet également de prévoir le cas où la liste est vide : 

    <ul>
    {% for skill in skills %}
        <li>{{ skill }}</li>
    {% else %}
        <li>Aucune compétence renseignée.</li>
    {% endfor %}
</ul>

    Une liste peut aussi contenir des dictionnaires. 

    projects = [
    {
        "name": "Mini-site personnel",
        "language": "Django"
    },
    {
        "name": "Quiz interactif",
        "language": "JavaScript"
    }
]
    {% for project in projects %}
    <article>
        <h2>{{ project.name }}</h2>
        <p>Technologie : {{ project.language }}</p>
    </article>
{% endfor %}

    Utiliser des filtres

    Un filtre permet de transformer une donnée transmise en Python avant de l'afficher.

    Il est placé après la variable avec le caractère |. 

    {{ my_variable | my_filter }}

    Quelques filtres courants :

    Filtre Code
    Majuscules 
    <p>{{ firstname | upper }}</p>
    Minuscules 
    <p>{{ firstname | lower }}</p>
    Première lettre en majuscules 
    <p>{{ firstname | capitalize }}</p>
    Nombre d'éléments dans une liste 
    <p>{{ skills | length }} compétences</p>
    Valeur si la variable n'est pas définie 
    <p>{{ city | default("Ville inconnue") }}</p>

    Erreurs fréquentes

    • Oublier les doubles accolades autour d'une variable : le nom de la variable apparaît comme du texte.
    • Oublier de fermer une condition avec endif : Jinja signale une erreur de syntaxe.
    • Oublier de fermer une boucle avec endfor : le template ne peut pas être interprété.
    • Parcourir une variable qui n'est pas une liste : le résultat peut être incorrect ou provoquer une erreur.
    • Écrire du code Python complet dans le template : Jinja utilise sa propre syntaxe simplifiée.
