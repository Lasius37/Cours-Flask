Projet : la gestion des fournisseurs

    Dans cette page

    • Énoncé
    • Conseils de réalisation

    Énoncé

    Vous devez réaliser une application Flask permettant de gérer les fournisseurs d'une entreprise.

    Gestion des données

    • Chaque fournisseur est renseigné dans une base de données.
    • Il est identifié par plusieurs éléments :
      • Nom de l'entreprise
      • Adresse
      • Ville
      • Numéro de téléphone
      • Adresse électronique
      • S'il préfère être contacté par téléphone ou par e-mail

    Fonctionnement

    • L'application comporte plusieurs pages :
      • une page d'accueil affichant la liste complète des fournisseurs
      • une page affichant toutes les informations sur un fournisseur
      • un formulaire de création d'un fournisseur
      • un formulaire de modification
      • un formulaire de suppression avec confirmation
    • Des liens permettent de naviguer dans l'ensemble du projet :
      • retour à l'accueil
      • sélection d'un fournisseur
      • ajout d'un fournisseur
      • modification d'un fournisseur (depuis l'accueil et depuis la page dédiée)
      • suppression d'un fournisseur (depuis l'accueil et depuis la page dédiée)
    • Sur la page d'accueil, on peut :
      • filtrer les fournisseurs par nom, par ville ou par préférence de contact
      • trier les fournisseurs par nom ou par ville

    Conseils de réalisation

    Organisation du projet

    • Structurer l'application.
    • Configurer une base de données.
    • Créer les dossiers et sous-dossiers nécessaires.
    • Créer les fichiers statiques nécessaires (style, js, images, etc.).

    Modèle et base de données

    • Créer le modèle Supplier, représentant un fournisseur et ses coordonnées :
      • id : son identifiant unique
      • name : son nom
      • address : son adresse
      • city : sa ville
      • phone : son numéro de téléphone
      • email : son adresse électronique
      • preferred_contact: sa préférence de contact (utiliser des boutons radio)

    Vues

    • Développer les fonctionnalités progressivement :
      • affichage de la liste complète des fournisseurs
      • affichage des informations détaillées d'un fournisseur
      • ajout d'un fournisseur
      • modification d'un fournisseur
      • suppression d'un fournisseur
      • filtre des fournisseurs selon leur nom, leur ville ou leur préférence de contact
      • tri des fournisseurs par nom ou par ville
    • Pour chacune, penser à :
      • développer sa vue
      • affecter une route
      • associer les bonnes méthodes (GET et/ou POST)

    Templates

    • Créer un template commun base.html contenant :
      • la structure générale de la page
      • un en-tête
      • un menu de navigation
      • un bloc destiné au contenu principal
      • un pied de page
      • le lien vers la feuille de style
    • Les autres templates héritent de cette structure :
      Template Contenu attendu
      suppliers.html Liste, filtrage et tri des fournisseurs
      supplier-detail.html Informations complètes sur un fournisseur
      create-supplier.html Formulaire vide de création
      update-supplier.html Formulaire prérempli de modification
    • Sur la page suppliers.html, pour chaque fournisseur, créer :
      • un lien vers sa page détaillée
      • un lien vers son formulaire de modification
      • une fonctionnalité de suppression avec confirmation
    • Sur la page supplier-detail.html, prévoyez :
      • le nom de l'entreprise
      • l'adresse complète
      • la ville
      • le numéro de téléphone
      • l'adresse électronique
      • la préférence de contact
      • un lien de modification
      • une fonctionnalité de suppression avec confirmation
      • un lien de retour vers la liste

    Formulaires

    • Utiliser les mêmes champs pour les formulaires de création et de modification :
      • nom de l'entreprise
      • adresse
      • ville
      • numéro de téléphone
      • adresse électronique
      • préférence de contact
    • Paramétrer correctement les champs :
      • type="text" pour les informations générales
      • type="tel" pour le téléphone
      • type="email" pour l'adresse électronique
      • required pour les champs obligatoires
      • maxlength pour limiter la longueur des textes
    • Utiliser l'attribut value pour afficher les données connues.
    • Pour la préférence de contact, utiliser l'attribut checked.

    Filtres et tris

    • Utiliser la méthode GET pour le formulaire de filtre.
    • Récupérer les valeurs envoyées avec GET avec request.args.
    • Proposer les champs de filtre :
      • un champ permettant de rechercher un fournisseur par son nom
      • un champ pour filtrer selon la ville
      • un champ pour filtrer selon la préférence de contact
    • Permettre à l'utilisateur de choisir le tri :
      • par nom de l'entreprise
      • par ville
    • Vérifier le choix effectué avant de filtrer ou de trier.
