    Créer une donnée

    Une ligne de la table est représentée en Python par un objet créé à partir du modèle.

    Pour créer un nouveau livre, instancier la classe Book en indiquant la valeur de chaque colonne : 

    book = Book(
    title="Le Petit Prince",
    year=1943,
    synopsis="Un aviateur rencontre un jeune prince venu d'une autre planète..."
)

    Il n'est pas nécessaire d'indiquer la valeur de id, car celle-ci sera générée automatiquement.

    À ce stade, l'objet existe uniquement dans le programme Python. Il n'est pas encore enregistré dans la base de données.

    Enregistrer une donnée

    Pour préparer l'enregistrement d'un objet, il faut l'ajouter à la session : 

    db.session.add(book)

    La session regroupe les modifications qui doivent être appliquées à la base de données.

    Ensuite, on valide ces modifications : 

    db.session.commit()

    L'enregistrement complet nécessite donc trois étapes : 

    book = Book(  # création de l'enregistrement
    title="Le Petit Prince",
    year=1943,
    synopsis="Un aviateur rencontre un jeune prince venu d'une autre planète..."
)

db.session.add(book)  # ajout de l'enregistrement à la session

db.session.commit()  # enregistrement de la session dans la base

    Après l'appel de commit(), le livre est enregistré durablement dans la table books.

    Pour effectuer cette opération directement au lancement du programme, utiliser le contexte de l'application : 

    with app.app_context():
    book = Book(
        title="Le Petit Prince",
        year=1943,
        synopsis="Un aviateur rencontre un jeune prince venu d'une autre planète."
    )

    db.session.add(book)

    db.session.commit()

    Erreurs fréquentes

    • Créer un objet sans l'ajouter à la session : la donnée n'est pas préparée pour être enregistrée.
    • Oublier db.session.commit() : les modifications ne sont pas enregistrées durablement.
    • Ajouter le même objet au lancement de chaque programme : une nouvelle ligne identique est créée à chaque exécution.
