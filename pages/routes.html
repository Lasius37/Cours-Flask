    Comprendre le rôle d'une route

    Une route associe une adresse de l'application à une fonction Python.

    Elle est créée avec le décorateur @app.route(), placé juste au-dessus de la fonction à exécuter. 

    @app.route("/")
def accueil():
    return "Bienvenue sur la page d'accueil"

    Lorsque l'utilisateur ouvre l'adresse http://127.0.0.1:5000/, Flask exécute la fonction accueil(). Le chemin "/" correspond à la racine du site, donc à sa page d'accueil.

    Créer plusieurs routes

    Une application contient généralement plusieurs routes. Chaque route possède une adresse et appelle une fonction différente. 

    @app.route("/")
def accueil():  # http://127.0.0.1:5000/ : page d'accueil
    return "Bienvenue sur la page d'accueil"

@app.route("/about")
def about():  # http://127.0.0.1:5000/about : page de présentation
    return "À propos de ce site"

@app.route("/contact")
def contact():  # http://127.0.0.1:5000/contact : page de contact
    return "Page de contact"

    Le nom de la fonction ne détermine pas l'adresse de la page. C'est le chemin écrit dans @app.route() qui définit l'URL.

    Créer une route dynamique

    Une route peut contenir une partie variable. Cette valeur est placée entre chevrons dans l'adresse. 

    @app.route("/user/<name>")
def user(name):
    return f"Bienvenue {name}"

    Dans cette route, la partie <name> peut prendre différentes valeurs.

    La valeur récupérée dans l'adresse est transmise à la fonction grâce au paramètre name.

    Le nom placé entre les chevrons doit être identique au nom du paramètre de la fonction.

    • /user/Pierre affiche Bienvenue Pierre
    • /user/Alice affiche Bienvenue Alice

    Erreurs fréquentes

    • Oublier le caractère / au début du chemin : la route n'est pas correctement déclarée.
    • Utiliser deux fois le même nom de fonction : Flask ne peut pas distinguer les deux fonctions.
    • Ouvrir une adresse qui ne correspond à aucune route : le navigateur affiche une erreur 404.
    • Utiliser un nom différent dans la route et dans le paramètre de la fonction : la valeur dynamique ne peut pas être transmise correctement.
