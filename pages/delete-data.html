    Créer la route de suppression

    La route contient l'identifiant de l'enregistrement à supprimer. 

    @app.route(
    "/books/<int:book_id>/delete",
    methods=["POST"]
)
def delete_book(book_id):
    # suppression du livre

    La route accepte uniquement la méthode POST. Par exemple, l'adresse suivante concerne le livre dont l'identifiant vaut 3 : 

    /books/3/delete

    La valeur 3 est transmise à la fonction sous le nom book_id.

    Supprimer la donnée

    Il faut d'abord récupérer l'enregistrement à partir de son identifiant. 

    book = db.session.get(
    Book,
    book_id
)

    Si aucun enregistrement ne correspond, la méthode renvoie None. 

    if book is None:
    return "Livre introuvable", 404

    La méthode db.session.delete() prépare ensuite la suppression de l'objet. 

    db.session.delete(book)

    La suppression est validée avec : 

    db.session.commit()

    Après la suppression, l'utilisateur est redirigé vers la liste des livres, par exemple. 

    return redirect(
    url_for("books")
)

    Créer le formulaire de suppression

    Dans le template books.html, chaque livre peut comporter un formulaire de suppression. 

    {% for book in books %}
    <article>
        ...

        <form
            action="{{ url_for(
                'delete_book',
                book_id=book.id
            ) }}"
            method="post"
        >
            <button type="submit">
                Supprimer
            </button>
        </form>
    </article>
{% endfor %}

    La fonction url_for() construit l'adresse à partir :

    • du nom de la fonction delete_book
    • de l'identifiant book.id

    Le formulaire utilise method="post" afin de déclencher la route de suppression. Il ne faut pas utiliser un simple lien <a>. Un lien envoie une requête GET, alors que la route de suppression accepte uniquement POST.

    Demander une confirmation

    Une suppression est définitive. Il est préférable de demander une confirmation avant l'envoi du formulaire. L'attribut onsubmit peut appeler la fonction JavaScript confirm(). 

    <form
    action="{{ url_for(
        'delete_book',
        book_id=book.id
    ) }}"
    method="post"
    onsubmit="return confirm('Supprimer ce livre ?')"
>
    <button type="submit">
        Supprimer
    </button>
</form>

    Une boîte de dialogue est affichée avant l'envoi :

    • si l'utilisateur confirme, le formulaire est envoyé
    • s'il annule, la requête n'est pas envoyée

    Cette confirmation améliore l'utilisation de l'application, mais elle ne remplace pas les vérifications réalisées dans la route Flask.

    Erreurs fréquentes

    • Oublier l'identifiant dans la route : l'application ne sait pas quel livre doit être supprimé.
    • Ne pas vérifier si book vaut None : l'application tente de supprimer un objet inexistant.
    • Utiliser un simple lien pour supprimer : le navigateur envoie une requête GET au lieu d'une requête POST.
    • Autoriser la méthode GET sur la route de suppression : une simple consultation de l'adresse peut supprimer une donnée.
    • Oublier method="post" dans le formulaire : la route de suppression n'accepte pas la requête reçue.
    • Oublier book_id dans url_for() : Flask ne peut pas générer l'adresse de suppression.
    • Oublier db.session.delete() : l'objet reste présent dans la base de données.
    • Oublier db.session.commit() : la suppression n'est pas enregistrée durablement.
    • Rediriger avant commit() : la suppression peut ne pas être enregistrée avant le changement de page.
    • Ne pas demander de confirmation : l'utilisateur peut supprimer une donnée par erreur.
