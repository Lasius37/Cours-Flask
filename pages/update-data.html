    Créer la route de modification

    Pour modifier un livre, il faut connaître l'identifiant de la donnée concernée.

    La route contient donc une partie variable : 

    @app.route(
    "/books/<int:book_id>/update",
    methods=["GET", "POST"]
)
def update_book(book_id):
    # modification du livre

    La variable book_id contient l'identifiant présent dans l'URL.

    Par exemple, l'adresse suivante permet de modifier le livre dont l'identifiant vaut 3 : 

    /books/3/update

    La route accepte deux méthodes :

    Méthode Rôle
    GET Afficher le formulaire de modification
    POST Recevoir les nouvelles valeurs

    Récupérer la donnée à modifier

    La méthode db.session.get() permet de récupérer le livre à partir de son identifiant. 

    book = db.session.get(
    Book,
    book_id
)

    Si aucun livre ne correspond à cet identifiant, la méthode renvoie None. 

    if book is None:
    return "Livre introuvable", 404

    Lorsque le livre existe, il peut être transmis au template contenant le formulaire. 

    return render_template(
    "update-book.html",
    book=book
)

    Préremplir le formulaire

    Le formulaire de modification doit afficher les valeurs déjà enregistrées.

    Préremplir un champ input

    L'attribut value permet de placer une valeur dans un champ input. 

    <label for="title">Titre</label>

<input
    type="text"
    id="title"
    name="title"
    value="{{ book.title }}"
    maxlength="100"
    required
>

    Le champ de l'année est prérempli de la même manière : 

    <label for="year">Année de parution</label>

<input
    type="number"
    id="year"
    name="year"
    value="{{ book.year }}"
    min="0"
    max="2100"
    required
>

    Préremplir une zone de texte

    Une balise textarea n'utilise pas l'attribut value. La valeur doit être placée entre les balises ouvrante et fermante : 

    <label for="synopsis">Synopsis</label>

<textarea
    id="synopsis"
    name="synopsis"
    rows="8"
    required
>{{ book.synopsis }}</textarea>

    Configurer la destination du formulaire

    Le formulaire doit transmettre l'identifiant du livre à la route de modification. 

    <form
    action="{{ url_for(
        'update_book',
        book_id=book.id
    ) }}"
    method="post"
>

    Modifier les valeurs

    Lorsque le formulaire est envoyé, la route reçoit une requête POST. Les nouvelles valeurs sont récupérées avec request.form. 

    if request.method == "POST":
    title = request.form["title"].strip()
    synopsis = request.form["synopsis"].strip()

    try:
        year = int(request.form["year"])

    except ValueError:
        return "L'année doit être un nombre entier."

    Les données sont ensuite vérifiées : 

    if title == "" or synopsis == "":
    return "Le titre et le synopsis sont obligatoires."

if year < 0 or year > 2100:
    return "L'année indiquée est incorrecte."

    Pour modifier la ligne existante, remplacer directement les attributs de l'objet book. 

    book.title = title
book.year = year
book.synopsis = synopsis

    Il ne faut pas créer un nouvel objet Book. Cela ajouterait une nouvelle ligne au lieu de modifier la ligne existante.

    Il n'est pas non plus nécessaire d'utiliser db.session.add(), car l'objet récupéré appartient déjà à la session.

    Les modifications sont validées avec : 

    db.session.commit()

    Après la modification, l'utilisateur est redirigé vers la liste des livres : 

    return redirect(
    url_for("books")
)

    Ajouter le lien de modification

    Dans le template books.html, chaque livre peut comporter un lien vers son formulaire de modification. 

    {% for book in books %}
    <article>
        ...

        <a href="{{ url_for(
            'update_book',  <-- le nom de la fonction de vue -->
            book_id=book.id  <-- la valeur de la partie variable -->
        ) }}">
            Modifier
        </a>
    </article>
{% endfor %}

    Erreurs fréquentes

    • Oublier l'identifiant dans la route : l'application ne sait pas quel enregistrement doit être modifié.
    • Ne pas vérifier si book vaut None : l'application tente de modifier un objet qui n'existe pas.
    • Oublier de transmettre book au template : le formulaire ne peut pas afficher les valeurs actuelles.
    • Oublier l'attribut value dans les champs input : les champs ne sont pas préremplis.
    • Utiliser value sur une balise textarea : la donnée n'est pas affiché.
    • Créer un nouvel objet au lieu de le modifier : une nouvelle ligne est ajoutée au lieu de modifier l'ancienne.
    • Oublier db.session.commit() : les nouvelles valeurs ne sont pas enregistrées.
    • Oublier la partie variable dans url_for() : Flask ne peut pas générer l'adresse de modification.
    • Traiter les données du formulaire lors d'une requête GET : aucune nouvelle valeur n'a encore été envoyée.
