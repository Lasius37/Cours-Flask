    Créer un template

    Écrire directement du code HTML dans une vue devient rapidement difficile à lire. Les pages HTML sont donc placées dans des fichiers séparés appelés templates.

    À la racine du projet, créer un dossier templates, puis un fichier html (pour l'exemple, index.html) dans ce dossier. 

    my_flask_project/
│
├── .venv/
├── templates/
│   └── index.html
│
├── app.py
└── requirements.txt

    Dans index.html, écrire une page HTML classique : 

    <!doctype html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <title>Accueil</title>
</head>
<body>
    <h1>Bienvenue sur mon site</h1>
    <p>Voici ma première page créée avec Flask.</p>
</body>
</html>

    Afficher un template

    Pour afficher un template, il faut importer la fonction render_template() depuis Flask. 

    from flask import Flask, render_template

    Dans la vue, remplacer le texte renvoyé par l'appel de la fonction render_template(). 

    @app.route("/")
def accueil():
    return render_template("index.html")

    Flask recherche automatiquement le fichier index.html dans le dossier templates.

    Créer plusieurs templates

    Chaque page de l'application peut utiliser son propre template. 

    templates/
├── index.html
├── about.html
└── contact.html

    Chaque route renvoie alors le template correspondant : 

    @app.route("/")
def accueil():
    return render_template("index.html")

@app.route("/about")
def about():
    return render_template("about.html")

@app.route("/contact")
def contact():
    return render_template("contact.html")

    Ces fichiers contiennent souvent des éléments identiques : la structure HTML, le menu, l'en-tête ou le pied de page. Copier ces éléments dans chaque fichier crée des répétitions. On utilise alors un template de base.

    Créer un template de base

    Un template de base contient les éléments communs à toutes les pages. Créer un fichier base.html dans le dossier templates. 

    <!doctype html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <title>Mon site Flask</title>
</head>
<body>
    <header>
        <h1>Mon site Flask</h1>

        <nav>
            <a href="/">Accueil</a>
            <a href="/about">À propos</a>
            <a href="/contact">Contact</a>
        </nav>
    </header>

    <main>
        {% block content %}
        {% endblock %}
    </main>

    <footer>
        <p>Mon premier site Flask</p>
    </footer>
</body>
</html>

    Le bloc content délimite la partie qui pourra être remplacée par le contenu propre à chaque page.

    Utiliser l'héritage d'un template

    Les autres templates peuvent hériter de la structure debase.html.

    Le fichier index.html devient : 

    {% extends "base.html" %}

{% block content %}
    <h2>Accueil</h2>
    <p>Bienvenue sur mon site.</p>
{% endblock %}
    • L'instruction extends indique que la page utilise base.html comme modèle.
    • Le contenu placé dans le bloc content remplace le bloc du même nom défini dans le template de base.

    Le fichier about.html peut également utiliser la même organisation : 

    {% extends "base.html" %}

{% block content %}
    <h2>À propos</h2>
    <p>Cette page présente mon site.</p>
{% endblock %}

    La structure commune est ainsi écrite une seule fois dans base.html.

    Erreurs fréquentes

    • Donner un autre nom au dossier templates : Flask ne trouve pas automatiquement les fichiers HTML.
    • Placer le dossier templates au mauvais endroit : une erreur TemplateNotFound peut être affichée.
    • Oublier d'importer render_template : Python ne reconnaît pas la fonction.
    • Utiliser un nom de fichier différent dans render_template() : Flask ne trouve pas le template demandé.
    • Oublier extends dans une page : le template de base n'est pas utilisé.
    • Utiliser un nom de bloc différent de celui défini dans base.html : le contenu n'apparaît pas à l'emplacement attendu.
