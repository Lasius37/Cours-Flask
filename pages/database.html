    Installer Flask-SQLAlchemy

    Flask ne possède pas directement les outils nécessaires pour gérer une base de données. L'extension Flask-SQLAlchemy permet de le faire de façon simple dans une application Flask.

    Dans le terminal de PyCharm, vérifier que l'environnement virtuel du projet est activé, puis installer Flask-SQLAlchemy : 

    pip install Flask-SQLAlchemy

    Mettre ensuite à jour le fichier des dépendances : 

    pip freeze > requirements.txt

    Configurer la base de données

    Dans le fichier app.py, importer Flask et Flask-SQLAlchemy : 

    from flask import Flask
from flask_sqlalchemy import SQLAlchemy

    Créer ensuite l'application Flask : 

    app = Flask(__name__)

    La clé de configuration SQLALCHEMY_DATABASE_URI indique à Flask-SQLAlchemy quelle base de données doit être utilisée. 

    app.config["SQLALCHEMY_DATABASE_URI"] = "sqlite:///my_db_file.db"

    Cette adresse se décompose ainsi :

    • sqlite indique que l'application utilise SQLite ;
    • my_db_file.db correspond au nom du fichier contenant la base de données.

    SQLite est adapté aux petites applications locales, car les données sont enregistrées dans un simple fichier.

    Avec cette configuration, Flask-SQLAlchemy placera le fichier my_db_file.db dans le dossier instance de l'application. 

    my_flask_project/
│
├── instance/
│   └── my_db_file.db
│
├── static/
├── templates/
│
├── app.py
└── requirements.txt

    Le fichier my_db_file.db n'existe pas encore. Il sera créé lorsque les premières tables seront générées.

    Initialiser Flask-SQLAlchemy

    Après avoir créé et configuré l'application Flask, créer l'objet db : 

    db = SQLAlchemy(app)

    L'objet db relie Flask-SQLAlchemy à l'application Flask.

    Il permettra notamment :

    • de créer les modèles de données
    • de générer les tables
    • d'enregistrer des données
    • de rechercher, modifier et supprimer des données.

    La configuration de la base doit être définie avant la création de l'objet db. 

    app = Flask(__name__)

app.config["SQLALCHEMY_DATABASE_URI"] = "sqlite:///my_db_file.db"

db = SQLAlchemy(app)

    Erreurs fréquentes

    • Installer Flask-SQLAlchemy en dehors de l'environnement virtuel : PyCharm peut ne pas trouver le module.
    • Oublier la clé SQLALCHEMY_DATABASE_URI : Flask-SQLAlchemy ne sait pas quelle base utiliser.
    • Créer l'objet db avant de configurer l'application : la configuration de la base n'est pas correctement prise en compte.
    • Utiliser un nom de fichier différent dans les exemples : il devient difficile d'identifier la base réellement utilisée.
    • Chercher immédiatement le fichier my_db_file.db : il ne sera créé qu'après la génération des premières tables.
