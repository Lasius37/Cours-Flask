Configurer une base de données

    Dans cette page

    • Installer et initialiser Flask-SQLAlchemy
    • Configurer la base de données
    • Relier la base à l'application
    • Erreurs fréquentes

    Installer et initialiser Flask-SQLAlchemy

    Dans le terminal de PyCharm, vérifier que l'environnement virtuel du projet est activé, puis installer Flask-SQLAlchemy : 

    pip install Flask-SQLAlchemy

    Mettre ensuite à jour le fichier des dépendances : 

    pip freeze > requirements.txt

    Dans app.py, importer Flask-SQLAlchemy et la classe DeclarativeBase. 

    from flask import Flask
from flask_sqlalchemy import SQLAlchemy
from sqlalchemy.orm import DeclarativeBase

    Créer ensuite une classe de base : 

    class DataBase(DeclarativeBase):
    pass

    Cette classe servira de base aux futurs modèles de l'application.

    Créer enfin l'objet db : 

    db = SQLAlchemy(model_class=DataBase)

    L'objet db donnera notamment accès :

    • à la classe utilisée pour créer les modèles ;
    • à la session permettant de manipuler les données ;
    • aux fonctions permettant de créer les tables.

    Configurer la base de données

    Créer ensuite l'application Flask : 

    app = Flask(__name__)

    La clé de configuration SQLALCHEMY_DATABASE_URI indique à SQLAlchemy quelle base de données utiliser. 

    app.config["SQLALCHEMY_DATABASE_URI"] = "sqlite:///my_db_file.db"

    Cette adresse se décompose ainsi :

    • sqlite indique le système de base de données utilisé ;
    • my_db_file.db correspond au nom du fichier qui contiendra les données.

    Avec cette configuration, le fichier my_db_file.db sera placé dans le dossier instance de l'application. 

    my_flask_project/
│
├── instance/
│   └── notes.db
│
├── static/
├── templates/
│
├── app.py
└── requirements.txt

    Le fichier notes.db n'est pas encore créé. Il apparaîtra lorsque les premières tables seront générées.

    Relier la base à l'application

    Après avoir configuré l'application, appeler la méthode init_app(). 

    db.init_app(app)

    La configuration de l'application doit être définie avant l'appel dedb.init_app(app). 

    app = Flask(__name__)

app.config["SQLALCHEMY_DATABASE_URI"] = "sqlite:///notes.db"

db.init_app(app)

    Erreurs fréquentes

    • Installer SQLAlchemy sans installer Flask-SQLAlchemy : l'import flask_sqlalchemy n'est pas reconnu.
    • Installer Flask-SQLAlchemy en dehors de l'environnement virtuel : PyCharm peut ne pas trouver le module.
    • Oublier la clé SQLALCHEMY_DATABASE_URI : Flask-SQLAlchemy ne sait pas à quelle base se connecter.
    • Configurer la base après db.init_app(app) : la nouvelle configuration n'est pas prise en compte.
    • Oublier d'appeler db.init_app(app) : la base n'est pas reliée à l'application Flask.
    • Chercher immédiatement le fichier notes.db : il ne sera créé qu'après la création des tables.
