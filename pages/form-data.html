    Autoriser les requêtes POST

    La route créée précédemment affiche le formulaire lorsqu'elle reçoit une requête GET. 

    @app.route("/books/create")
def create_book():
    return render_template("create-book.html")

    Cependant, le formulaire utilise la méthode POST : 

    <form
    action="{{ url_for('create_book') }}"
    method="post"
>

    La route doit donc être configurée pour accepter les deux méthodes : 

    @app.route(
    "/books/create",
    methods=["GET", "POST"]
)
def create_book():
    return render_template("create-book.html")
    Méthode Rôle
    GET Afficher le formulaire
    POST Recevoir les données envoyées par le formulaire

    Identifier la méthode reçue

    La même fonction de vue peut recevoir une requête GET ou POST. Pour connaître la méthode reçue, il faut importer l'objet request : 

    from flask import render_template, request

    La propriété request.method contient le nom de la méthode HTTP utilisée. 

    if request.method == "POST":  # traiter les données envoyées

    Lorsque la méthode est GET, la condition n'est pas exécutée et le formulaire est simplement affiché. 

    @app.route(
    "/books/create",
    methods=["GET", "POST"]
)
def create_book():
    if request.method == "POST":
        # traitement du formulaire

    return render_template("create-book.html")

    Récupérer les données du formulaire

    Les valeurs envoyées par un formulaire utilisant POST sont accessibles dans request.form.

    Chaque valeur est identifiée par l'attribut name de son champ HTML. 

    <input
    type="text"
    id="title"
    name="title"
>

    La valeur de ce champ peut être récupérée ainsi : 

    title = request.form["title"]

    Les trois valeurs du formulaire sont donc récupérées avec : 

    title = request.form["title"]
year = request.form["year"]
synopsis = request.form["synopsis"]

    Le nom placé entre crochets doit être exactement identique à la valeur de l'attribut name correspondant.

    Champ HTML Récupération dans Flask
    name="title" request.form["title"]
    name="year" request.form["year"]
    name="synopsis" request.form["synopsis"]

    Convertir les valeurs

    Les données récupérées depuis un formulaire sont reçues sous la forme de chaînes de caractères.

    Le champ year utilise pourtant le type Integer dans le modèle : 

    year = db.Column(
    db.Integer,
    nullable=False
)

    Il faut donc convertir la valeur reçue avec int() : 

    year = int(request.form["year"])

    Les valeurs textuelles n'ont pas besoin de conversion : 

    title = request.form["title"]
synopsis = request.form["synopsis"]

    Une conversion incorrecte provoque une erreur. Par exemple, la fonctionint() ne peut pas convertir un texte qui ne représente pas un nombre entier.

    Vérifier les données

    Les attributs HTML comme required, min et max améliorent la saisie dans le navigateur. Cependant, les données doivent également être vérifiées dans Flask.

    Supprimer les espaces inutiles

    La méthode strip() supprime les espaces placés au début et à la fin d'un texte. 

    title = request.form["title"].strip()
synopsis = request.form["synopsis"].strip()

    Vérifier les champs obligatoires

    if title == "" or synopsis == "":
    return "Le titre et le synopsis sont obligatoires."

    Vérifier l'année

    if year < 0 or year > 2100:
    return "L'année indiquée est incorrecte."

    Enregistrer les données

    Après avoir récupéré et vérifié les valeurs, on crée un nouvel objet Book. 

    book = Book(
    title=title,
    year=year,
    synopsis=synopsis
)

    On ajoute ensuite l'objet à la session : 

    db.session.add(book)

    Enfin, on enregistre définitivement la modification : 

    db.session.commit()

    Le traitement complet de l'enregistrement est donc : 

    book = Book(
    title=title,
    year=year,
    synopsis=synopsis
)

db.session.add(book)
db.session.commit()

    Après l'enregistrement, la fonction peut renvoyer un message provisoire : 

    return "Le livre a été enregistré."

    Erreurs fréquentes

    • Oublier d'ajouter "POST" dans l'argument methods : Flask refuse l'envoi du formulaire.
    • Oublier d'importer request : l'application ne peut pas accéder aux informations de la requête.
    • Utiliser une clé différente de l'attribut name du champ : Flask ne trouve pas la valeur attendue dans request.form.
    • Oublier de convertir l'année avec int() : la valeur reste une chaîne de caractères.
    • Convertir directement une valeur incorrecte sans gérer ValueError : l'application renvoie une erreur serveur.
    • Faire confiance uniquement aux contrôles HTML : des données incorrectes peuvent tout de même être envoyées à Flask.
    • Créer l'objet Book avant d'avoir vérifié les données : une valeur incorrecte peut être préparée pour l'enregistrement.
    • Oublier db.session.add() : le nouvel objet n'est pas ajouté à la session.
    • Oublier db.session.commit() : le livre n'est pas enregistré durablement.
    • Exécuter le traitement du formulaire pour une requête GET : l'application cherche des valeurs qui n'ont pas encore été envoyées.
